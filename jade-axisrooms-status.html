Jade PMS + Axis Rooms — Status & Dashboard Audit

Updated 2026-07-04 · Based on dashboard audit, security audit, and current Axis Rooms code paths

Executive summary: The Axis Rooms surface inside the Jade dashboard is functionally built but still in partial rollout. The audit confirms that 12 of 15 Axis APIs are implemented in Jade, including outbound inventory/price sync, inbound webhook intake, pull reconciliation, connected OTAs, verify tools, and per-OTA pause/resume. The dashboard itself is operational; the remaining work is Axis-side mapping, webhook registration, scheduler setup, and sandbox / production UAT.

Confirmed working in Jade

  • Axis settings panel loads mappings, connection metadata, connected OTAs, verify tools, and sync logs.
  • Outbound sync code exists for inventory, price, and restrictions on mapped villas.
  • Inbound webhook route accepts POST payloads, validates accessKey, and writes booking changes into Jade.
  • Conflict handling, audit logging, and staff email notifications are wired for OTA overlaps.

What still blocks full rollout

  • Sandbox hotel mapping in Axis Channel Manager is not active for the test property.
  • API 9 still needs registration from Axis and live confirmed / cancelled sample payloads.
  • Scheduled retry / pull protection requires CRON_SECRET plus Vercel Pro or an external cron for sub-daily jobs.
  • Portfolio villas should stay website_only until each mapping passes UAT and is intentionally flipped to channel_managed.
