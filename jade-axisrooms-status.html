  • Calendar + occupancy grid
  • Staff manual booking → on hold (external payment)
  • Booking Records list + filters
  • Booking folio + full activity history
  • Manual blocks (owner holds)
  • Housekeeping stay status
  • Conflict queue UI
  • Public /book + Razorpay direct bookings

  • Axis Rooms settings + CSV export for onboarding

5. What Axis Rooms does NOT provide

These are not missing Jade work — Axis has no REST API for them. Jade uses workarounds:

Business needAxis API?Jade approach
Push a reservation to OTAs (create booking on Airbnb/Booking.com) No API API 1 — set free: 0 on nights (blocks inventory, not a reservation record)
On-hold / partial payment on channel manager No API Staff hold → API 1 close immediately (Option B); confirm in folio only
Create property on Axis via REST No API CSV to Axis team → map IDs in Jade dashboard
Refund OTA or direct guest via Axis No API Razorpay refunds in Jade for website bookings; OTA refunds handled by OTA/Axis
Housekeeping / stay operations No API Jade PMS only — stayStatus on booking
Cancel OTA booking from Jade (outbound cancel reservation) No API API 1 free: 1 releases inventory; OTA cancel arrives via API 9 inbound
