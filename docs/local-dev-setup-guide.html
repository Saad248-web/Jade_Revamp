What this project needs locally

Why the admin page showed “database unavailable”

Jade ReVamp is a Next.js site. The Admin Dashboard (/admin) manages villa bookings stored in PostgreSQL.

The previous developer likely had Postgres on their machine or used a remote server. Your PC has no database running yet, so login fails with HTTP 500 — not because the password is wrong.

Recommended path: Install Docker Desktop once, then use one command to create a brand-new empty database. You do not need to recover the old developer’s data for learning.

Install Docker Desktop

Your turn — one-time install

  1. Download: Docker Desktop for Windows
  2. Install and restart your PC if prompted.
  3. Start Docker Desktop and wait until it says Engine running.
Docker is a small “virtual computer” that runs PostgreSQL for you. You do not need to install Postgres separately if you use Docker.

Create a fresh local database

Your turn — deletes any old local Docker data

This creates a new empty database with all tables. Safe for learning.

cd C:\Users\Admin\Desktop\Jade_ReVamp
npm run db:reset

First run may take 1–3 minutes (downloads Postgres image).

What db:reset does: removes old Docker volume → starts Postgres → runs schema.sql automatically.

Troubleshooting

Symptom Fix
“Database unavailable” on login Docker not running → start Docker Desktop → npm run db:up
docker not recognized Install Docker Desktop; restart PowerShell
Port 5432 already in use Another Postgres installed — stop it or change port in docker-compose.db.yml
Incorrect password (401) Use password from .env.local → ADMIN_PASSWORD
Want empty DB again npm run db:reset
