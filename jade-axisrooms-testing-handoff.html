Jade Host PMS — Axis Rooms Integration Reference

PMS: Jade Host PMS · UAT environment: https://jade-revamp.vercel.app · Document date: 9 July 2026 (UAT session — inbound validation & inventory flows verified locally)

Integration scope: Jade Host PMS is a whole-villa booking system. Each bookable villa is one inventory unit (availability 0 or 1 per night — not multiple rooms inside a property). Outbound inventory, price, and restriction sync uses Axis APIs 1, 2, 6, 7, and 15. Inbound OTA bookings are received on API 9. 1. Inventory model — villa as the unit

Jade sells entire villas, not individual rooms within a villa. Axis field names use roomId / roomTypeId; in Jade each mapped villa corresponds to one such ID with per-night availability of 0 (booked) or 1 (available).

Business eventJade calendarAPI 1 value sent
Villa booked (confirm / hold) Night locks acquired on stay nights free: 0 per night
Booking cancelled Night locks released free: 1 on current date range
Booking dates changed (staff) Old locks released, new locks acquired free: 1 on old range, then free: 0 on new range
OTA booking modify (API 9 inbound) Local night-lock swap only No outbound push (CM owns OTA inventory)
Night model: check-out is exclusive (same-day turnover allowed).
Example: stay 21 Oct → 22 Oct blocks the night of 21 Oct only; 22 Oct is checkout day.

3. Per-villa mapping (stored in Jade dashboard)

Each villa record stores Axis identifiers and a channel state:

FieldDescription
axisRooms.propertyIdAxis hotelId for this villa
axisRooms.roomTypeIdAxis roomId — whole villa as one inventory unit
axisRooms.ratePlanIdAxis rate plan for price sync
axisRooms.ratePlanNameAxis rate plan display name (CSV export)
channelModewebsite_only (no outbound sync) or channel_managed (sync active when IDs are valid)

Property mappings are viewable and exportable from the Axis Rooms settings screen in the Jade dashboard. CSV export is available for property onboarding.

same dates		 Restore inventory UAT pending API 1 free: 1 on booking’s current checkIn / checkOut range. Failed pushes retried automatically.
2. Booking modified

2b. Modified booking cancelled
24/10–26/10		 Restore inventory on current dates UAT pending Cancel opens inventory on the booking’s current (post-modify) date range.

API 1 triggers summary

  • Close (free: 0): payment confirmed, staff hold placed, staff date change (new range), calendar block created
  • Open (free: 1): staff cancel, calendar block released, staff date change (old range)
  • Modify sequence: open old nights → close new nights (direct / staff / website bookings only)
