2b. How dashboard edits reach the public site

One save path for both editors. Quick Edit and the 13-step Full Property Editor both call PATCH /api/dashboard/villas/[slug]. Data is stored in MongoDB. The public API (GET /api/public/villas) merges Mongo with the coded portfolio catalogue — dashboard values win when staff have set them; coded JSON fills gaps (descriptions, amenities) when Mongo fields are empty.
You edit in dashboardQuick EditFull EditorPublic surfaces updated
Name, location, thumbnail, stats✓✓/villas cards, detail hero, SEO
Pricing, pax, tax, wedding tiers✓✓ (step 13)Cards, detail pricing, /book quotes
Description, amenities, spaces, FAQ, video—✓Villa detail page sections
Visibility / bookable✓✓Directory listing, Book vs Enquire CTA

After save, revalidatePath busts Next.js cache for /villas, /villas/[id], and /book. Portfolio catalogue villas (Diamond, Magnolia, …) cannot be deleted — use Hidden visibility instead.

6. Security & quality

Security strengths

  • Role-based access on every dashboard API (admin, staff, team, SEO, dev)
  • Mongo user re-validated on each request (suspended accounts blocked immediately)
  • Rate limiting on public booking and lead endpoints
  • Razorpay webhook HMAC verification + idempotency store
  • Night-lock concurrency prevents double-booking same dates
  • Audit log on villa/booking/user mutations
  • Guest PII erasure endpoint (DPDP)
  • Dashboard pages noindex — not indexed by search engines

Items to address before go-live

  • Rotate seed password — default JadeHost2026! must not be used in production
  • No MFA — consider for admin accounts (planned enhancement)
  • Cron jobs — CRON_SECRET + scheduler; sub-daily expire/retry need Vercel Pro or external cron; daily pull/blogs in vercel.json
  • Resend — verify sending domain; set notify emails for bookings, leads, careers, conflicts
  • Full booking E2E — book → pay → webhook → confirmed not yet run on production URL
  • Axis certification — sandbox UAT, API 9 registration, real inbound JSON samples from Axis
  • Channel mode — flip portfolio villas to channel_managed only after mapping UAT

Automated test suite (2026-07-01): 31 files · 187 tests · all passing — includes leads source validation, email recipient parsing, Axis channel state, connected OTAs, villa media resolver, merge/public text, pricing display, editor validation, deletion guards, RBAC registry, booking forms, payment service.
