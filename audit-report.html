What’s done (high confidence)

  • Modern foundation: Next.js 14 App Router + strict TypeScript + Tailwind.
  • Core routes shipped: marketing pages, villas, blogs, booking, legal pages.
  • SEO stack: robots.ts, sitemap.ts, metadata base, JSON-LD helpers, IndexNow-ready API.
  • GEO / LLM layer (elite): public/llms.txt (assistant routing + anti-hallucination hints); mirrored robots.txt rules for GPT/OAI/Claude/Perplexity/Google-Extended crawlers; every villa + /spaces + legal URLs in sitemap; per-villa VacationRental + title/description/OG/Twitter/canonical from server layouts; WebSite + enriched Organization/LodgingBusiness graph; /wishlist + /admin noindex aligned with crawler blocks.
  • Bookings backend: PostgreSQL pool + create booking + availability + admin listing.
  • Leads & hiring: general + wedding + Rathaa enquiries to POST /api/leads; partner photos to POST /api/leads/partner; careers to POST /api/careers/apply — each rate-limited.
  • Transactional email: booking create can email guest/staff via Resend when env keys + BOOKING_NOTIFY_EMAIL (and related) are set.
  • GA4 ready: loads from providers.tsx when NEXT_PUBLIC_GA_ID is present.
  • Razorpay + book pay: server order, success-screen Checkout with booking_uuid, webhook signature verification — ops still sets dashboard URL, secrets, and production DB migration.
  • IndexNow: `/api/indexnow` implemented; key file in `public/`. Production requires INDEXNOW_API_SECRET + Authorization: Bearer …; URLs must be https on configured host.
  • Security baseline (2026-05-01): src/middleware.ts on /api/* — verb allowlist, 600 requests/min/IP coarse cap, CORP; hardened headers in next.config.mjs (prod HSTS, COOP, Permissions-Policy); safe JSON limits; timing-safe admin secret; booking villaId allowlist + shared validation with UI.
