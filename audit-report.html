What’s done (high confidence)

  • Modern foundation: Next.js 14 App Router + strict TypeScript + Tailwind.
  • Core routes shipped: marketing pages, villas, blogs, booking, legal pages.
  • SEO stack: robots.ts, sitemap.ts, metadata base, JSON-LD helpers, IndexNow-ready API.
  • GEO / LLM layer (elite): public/llms.txt (assistant routing + anti-hallucination hints); mirrored robots.txt rules for GPT/OAI/Claude/Perplexity/Google-Extended crawlers; every villa + /spaces + legal URLs in sitemap; per-villa VacationRental + title/description/OG/Twitter/canonical from server layouts; WebSite + enriched Organization/LodgingBusiness graph; /wishlist + /admin noindex aligned with crawler blocks.
  • Bookings backend: PostgreSQL pool + create booking + availability + admin listing.
  • Leads & hiring: general + wedding + Rathaa enquiries to POST /api/leads; partner photos to POST /api/leads/partner; careers to POST /api/careers/apply — each rate-limited.
  • Transactional email: booking create can email guest/staff via Resend when env keys + BOOKING_NOTIFY_EMAIL (and related) are set.
  • GA4 ready: loads from providers.tsx when NEXT_PUBLIC_GA_ID is present.
  • Razorpay scaffolding: server order creation, booking UUID carried in Razorpay notes when sent, webhook verifies signature and marks payment state — needs dashboard URL + secrets on your host.
  • IndexNow: `/api/indexnow` implemented; key file in `public/`. Production requires INDEXNOW_API_SECRET + Authorization: Bearer …; URLs must be https on configured host.
  • Security baseline (2026-05-01): src/middleware.ts on /api/* — verb allowlist, 600 requests/min/IP coarse cap, CORP; hardened headers in next.config.mjs (prod HSTS, COOP, Permissions-Policy); safe JSON limits; timing-safe admin secret; booking villaId allowlist + shared validation with UI.

What’s missing / risky (prioritized)

  • P0 link/asset 404s: fixed (internal scan now reports 0 broken routes/assets).
  • Redirect coverage: policy 301s + /blog/* → /blogs/* added; other legacy URL migrations still need mapping.
  • Security — next tier: elite baseline is in (middleware + headers + API validation + timing-safe admin). For scale: move rate limits to Redis/Upstash; optional JWT/session admin + CSP nonces; full monitoring (e.g. Sentry).
  • Performance at scale: large `public/` media library may benefit from CDN/object storage.
  • SEO polish: ship a dedicated 1200×630 global OG (“og-default.jpg”); villas already use hero imagery for shares. Off-site citations (Reddit, YouTube, Wikipedia-class) remain a separate growth track.
  • Commercial / funnel APIs (large uplift): general + wedding enquiries, Rathaa overlay, Partner multipart, and Careers apply now hit real API routes (/api/leads*, /api/careers/apply) with notifications when Resend is configured. Booking path unchanged (Postgres).
  • Payments checkout wire: Razorpay server order, webhook verification, and booking UUID notes are implemented — the book UI must still pass booking_uuid post-create + open Checkout; align payment_gateway_state vs status operationally.
  • GA4 wiring: GoogleAnalytics is mounted via providers.tsx when NEXT_PUBLIC_GA_ID is set.
  • QA & resilience: Vitest + one booking validation suite; src/app/error.tsx and global-error.tsx exist — still expand Playwright/coverage & Sentry-style monitoring for production-grade confidence.
